# Complete Documentation Index

**ChangeGuard AI — Predictive Risk Scoring for Change Advisory Board Decisioning**

> A single navigable index of every artifact in this case study, organized by project phase rather than file number, for readers who want to follow the work in the order it actually happened. See the [README](./README.md) for the project overview and outcomes summary.

## How to Use This Index

This repository documents the full lifecycle of an AI/ML project, from the first discovery conversation through six months of post-launch governance. The 15 documents below are listed in delivery order, grouped by the seven stages they belong to, with a one-line note on what each one is for and why it exists.

If you only have a few minutes: read the [README](./README.md), then the [Project Closure Report](./14-Project-Closure-Report.md), then [Lessons Learned](./15-Lessons-Learned.md).

## A Note on How This Documentation Was Built

Every document in this repository traces back to a specific decision, finding, or conversation described in the narrative case study — nothing here is generic template filler. Where a number, a stakeholder concern, or a risk appears in one document, it is cross-linked to the document where that item was first raised or where it was ultimately resolved, so a reader can follow any thread (for example, the team-identity fairness risk) from its first mention in the [Project Charter](./01-Project-Charter.md), through the [Risk Register](./08-Risk-Register.md), the [Sprint Plan](./09-Sprint-Plan.md), and into the [Lessons Learned](./15-Lessons-Learned.md) reflection on it.
